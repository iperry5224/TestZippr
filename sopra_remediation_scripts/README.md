# SOPRA Remediation Scripts

**These are the actual PowerShell scripts SOPRA uses to remediate failed security controls.**

## What are these?

When a control fails an assessment (e.g., "Domain Admin Account Security"), SOPRA generates a runnable PowerShell script to fix it. Each `.ps1` file corresponds to one of the 200 security controls in SOPRA's library.

- **Purpose:** Remediate (fix) failed controls
- **Format:** PowerShell (.ps1)
- **Source:** `sopra_controls.py` — control definitions with step-by-step remediation commands
- **Total:** 200 scripts across 20 categories (Active Directory, Network, Endpoint, etc.)

## Example: AD_001.ps1 (Domain Admin Account Security)

```powershell
# Remediation Script for AD-001: Domain Admin Account Security
# Family: Access Control
# Generated by OPRA

# Step 1: Audit current Domain Admin membership
Get-ADGroupMember -Identity 'Domain Admins' | Select-Object Name, SamAccountName, Enabled | Export-CSV DomainAdmins.csv

# Step 2: Remove unnecessary accounts from Domain Admins
Remove-ADGroupMember -Identity 'Domain Admins' -Members '<username>' -Confirm:$false
```

## How to use

1. Run an assessment in SOPRA to identify failed controls.
2. In the Reports or Remediation tab, click a failed control.
3. SOPRA shows the remediation script — or use these exported files directly.
4. Review, customize placeholders (e.g., `<username>`), and run in your environment.
